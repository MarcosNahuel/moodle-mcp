├── curso/                # crear / actualizar / duplicar / archivar / listar / obtener_contexto

├── alumnos/              # listar / matricular_csv / dar_baja / crear_grupo / asignar_a_grupo /

Each family owns its own subfolder with:
- `<tool>.ts` per facade.
- `_schemas.ts` per family for shared Zod schemas.
- Mirror folder in `tests/unit/<familia>/` with `.test.ts` per facade + `_integration.test.ts`.

## Non-negotiable principles (repeated from design doc)

1. Idempotency by `idnumber` on every write (`mcp:<type>:<hash>`).
2. Zod validation before every WS call. No `any`.
3. Typed errors: `MOODLE_PLUGIN_MISSING`, `MOODLE_WS_TIMEOUT`, `MOODLE_INVALID_TOKEN`, `MOODLE_PERMISSION_DENIED`, `MOODLE_WS_EXCEPTION`.
4. Exponential backoff retry on 5xx and timeouts; no retry on 4xx.
5. Rate limit 10 req/s (configurable).
6. Each facade: unit test with `nock` + 1 E2E integration test against `bitnamilegacy/moodle:5.0.2` + plugin companion.
7. Default `oculto` mode for any new-content publication.
8. Domain-agnostic: the MCP never assumes Italicia/Alicia. README and examples do mention Italicia as primary consumer.
9. Open-source MIT.
10. Node 20+ target, Node 24 LTS dev.

## Versioning rule for v0.5.x

- `v0.5.0`: first release with all 44 new facades (the 5 existing + `generate_video` already shipped).
